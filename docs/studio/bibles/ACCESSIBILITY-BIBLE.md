# Accessibility Bible v0.1

**Owner:** UX Lead  
**Status:** Draft — seeded from POS v3.0 Part I §1.8  
**Pillars served:** P4 (Fair Challenge), P5 (Collectible Delight)

---

## 1. Vision

Everyone can enjoy the game. Accessibility is **designed in**, not bolted on at ship. Assist options preserve dignity and player agency.

## 2. Principles

1. **Options, not lesser modes** — Assist is a player choice, not a stigma.
2. **Separate metrics** — Assist-on playtests tracked separately from standard.
3. **Motion safety** — Camera comfort is a launch requirement, not post-launch.
4. **Input equity** — Full remapping, hold/toggle options, generous timing buffers.
5. **Readable defaults** — Contrast and color work without modes first.
6. **Cognitive respect** — Hints optional; never condescending frequency.

## 3. Standards

| Category | Requirement | Milestone |
|----------|-------------|-----------|
| Input | Remappable controls, hold/tap toggle | M7 |
| Visual | Colorblind palettes, high contrast mode | M8 |
| Motion | Shake reduction, FOV stability, comfort mode | M3 |
| Cognitive | Optional markers, hint cadence control | M6 |
| Difficulty | Assist: extra coyote, damage reduction, invincibility toggle | M7 |
| Audio | Visual SFX cues for critical feedback | M8 |
| UI | Scalable text, screen reader hooks for menus | M8 |

**Certification:** UX Lead sign-off required at G3 and G4.

## 4. Patterns

- **Comfort camera profile** — Reduced shake, capped angular velocity
- **Assist ladder** — Coyote+ → damage reduce → invincibility (player picks depth)
- **Visual telegraph duplicate** — Enemy wind-up shown in shape + icon

## 5. Anti-patterns

- Assist hidden in debug menu
- Single accessibility pass at end of production
- Penalizing assist users in leaderboards
- Motion sickness ignored in camera tuning
