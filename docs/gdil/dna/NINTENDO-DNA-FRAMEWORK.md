# Nintendo DNA Framework

**Subsystem:** GDIL §10  
**Purpose:** Abstract, timeless design principles from highest-quality 3D platformers — **no copyrighted content**

*This framework captures transferable design philosophy, not assets, characters, or proprietary mechanics.*

---

## 1. Design Rules

### DNA-01: Readability First

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Player success depends on reading space faster than reacting to surprises |
| **Measurable indicators** | Jump success ≥75%; occlusion <15%; hazard survey fair ≥70% |
| **Validation** | Screenshot greyscale test; camera occlusion metric; playtest readability |
| **Failure signals** | "I didn't see it" deaths >30%; camera intervention >3/sec |

### DNA-02: Mechanical Clarity

| Aspect | Specification |
|--------|---------------|
| **Rationale** | One mechanic introduced at a time; each has a clear verb |
| **Measurable indicators** | TEACH 0 deaths; survey mechanic understanding ≥70% |
| **Validation** | Grammar audit; MKB tutorial strategy review |
| **Failure signals** | Multiple new mechanics in one TEACH; confusion survey >30% |

### DNA-03: Layered Mastery

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Main path accessible; mastery routes reward skill expression |
| **Measurable indicators** | Optional route uptake ≥10%; competence slope positive |
| **Validation** | MASTER grammar present; route telemetry |
| **Failure signals** | Single dominant path >95%; no MASTER nodes |

### DNA-04: Joyful Movement

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Movement itself is pleasurable before goals exist |
| **Measurable indicators** | Jump satisfaction ≥7/10; expression driver ≥0.5 |
| **Validation** | Character Feel Lab presets; movement survey |
| **Failure signals** | Movement described as "sluggish" or "floaty" >40% |

### DNA-05: Surprise Through Interaction

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Delight from systems colliding, not random events |
| **Measurable indicators** | Positive surprise survey; interaction novelty events |
| **Validation** | Interaction Matrix unexpected combos documented as delights |
| **Failure signals** | Negative surprise (unfair deaths) > positive surprise |

### DNA-06: Dense But Understandable Worlds

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Rich worlds with clear visual hierarchy — not empty, not cluttered |
| **Measurable indicators** | Exploration index 0.35–0.55; vista dwell >3s |
| **Validation** | Art readability review; World Identity visual dimension |
| **Failure signals** | Players lost >20% survey; or linear-only exploration <0.2 |

### DNA-07: Fair Failure

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Death teaches; retry is fast; player owns the mistake |
| **Measurable indicators** | Recovery ≥90%; frustration <0.25; fair death survey ≥70% |
| **Validation** | Death cause taxonomy; respawn timer |
| **Failure signals** | Same-spot death clusters; quit after death >25% |

### DNA-08: Reward Cadence

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Frequent micro-rewards sustain engagement between peaks |
| **Measurable indicators** | Reward density 8–15/min; first reward <60s |
| **Validation** | Reward timeline chart per level |
| **Failure signals** | >90s reward desert; density <5/min |

### DNA-09: Curiosity Over Checklist

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Optional content feels discovered, not assigned |
| **Measurable indicators** | Secret discovery 40%+ obvious; backtrack ratio healthy |
| **Validation** | Secret taxonomy audit; exploration heatmap |
| **Failure signals** | 100% completion feels grindy; <20% secret discovery |

### DNA-10: Climax With Breath

| Aspect | Specification |
|--------|---------------|
| **Rationale** | Bosses and peaks need recovery windows and celebration |
| **Measurable indicators** | Boss recovery ≥1.5s; celebration reward <2s; emotional triumph ≥70% |
| **Validation** | Boss Intelligence framework checklist |
| **Failure signals** | Boss deaths >4 avg; no post-defeat breathing room |

## 2. DNA Compliance Score

```
DNA Score = (rules_passing / 10) × 100

Minimum: 70 for GDIL authorization
Minimum: 85 for world ship
Minimum: 90 for commercial ship (M9)
```

## 3. Validation Process

| Cadence | Activity |
|---------|----------|
| Per mechanic approval | DNA-02, DNA-04 check |
| Per level | DNA-01, DNA-06, DNA-08 |
| Per boss | DNA-07, DNA-10 |
| Per world | Full 10-rule audit |
| Pre-ship | DNA score ≥90 |

## 4. Integration

| System | Role |
|--------|------|
| Fun Engine | DNA rules map to driver floors |
| Mechanic Lifecycle | DNA check at Approval stage |
| World Identity | DNA-06 per world |
| Boss Intelligence | DNA-10 |
| GDIL Dashboard | DNA compliance widget |

## 5. Anti-Copying Statement

This framework references **principles** observed across the genre. It does not reproduce:
- Character designs, names, or likenesses
- Proprietary mechanic names or exact parameter sets
- Level layouts, music, or art assets
- Trademarked terminology

All implementation must be **original expression** guided by abstract rules.
